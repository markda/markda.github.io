Eres un git sucio

"Remove everything that has no relevance to the story. If you say in the first chapter that there is a rifle hanging on the wall, in the second or third chapter it absolutely must go off. If it's not going to be fired, it shouldn't be hanging there."

Anton Chekhov

Assuming that git is already installed on the server, but not on your local machine, the first thing we need to do is install git. The following instructions are for Debian-based distributions. For installing git on other Linux distributions, look here.


$ sudo apt-get install git


The next thing we need to do is create a directory that will be used as your git repository. It doesn't matter a great deal where you create because later on you can copy it or clone it elsewhere. Once it is created, we need to initialise it as a git repository:


$ mkdir project_name
$ cd project_name
$ git init


Then we need to clone it into a bare repository otherwise you won't be able to push and pull updates from the server.


$ cd ../
$ git clone --bare project_name project_name.git


At this point you will probably get the following warning: You appear to have cloned an empty repository. You can ignore it because currently it's true. Once we have the bare repository, which is just the name of your original directory with a .git suffix, we need to copy it to the server. Here the address of the server is xxxx.xx.xx.xx. This will obviously be something more reasonable for the actual address. You will need a user account and access rights to write to /opt/git. Here the username is simply user. You will be prompted to enter your password for accessing the server via ssh.


$ scp -r project_name.git user@xxxx.xx.xx.xx:/opt/git/


At this point we're going to do something that might seem unnecessary, but it checks if other people who might be working in the same repository can clone it and work with it too:


$ rm -r project_name project_name.git
$ git clone user@xxxx.xx.xx.xx:/opt/git/project_name.git


Again, you'll likely get a warning about your vacuous repository: You appear to have cloned an empty repository. Ignore this. We'll add something to it now to check that it has been set up correctly:


$ cd project_name
$ touch test.txt
$ git add test.txt
$ git commit -m "Testing git repo."
$ git push origin master


You should now have a repository up and running on your server that anyone with access rights can work on.
