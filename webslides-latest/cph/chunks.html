"Chunk-and-Pass" Processing

Christiansen and Chater, 2016

  • Working-memory restrictions.

    "Now-or-Never" bottleneck.

  • Need to compress and recode linguistic input into a hierarchical abstract representation.

  • For sentence-level NLP analyses, this means using a chunker.

Chunking

A sequence labelling task.

Each word is labelled, B, I, or O.

  • B

    A token that begins a chunk.

    Suffixed with chunk phrase type.

    E.g. B-NP for a noun phrase.

  • I

    A token inside of a chunk.

    Also suffixed with chunk phrase type.

    E.g. I-VP for a verb phrase.

  • O

    Anything outside of a chunk.

System Flow

  • 1. Chunker

    I. Supervised chunker classifies each word (IOB notation).

    II. Sends chunks to the Inside parser.

    III. Sends chunk embeddings to the Outside parser.

  • 2. Inside Parser

    Faster parser for simpler, shorter relations.

    Parsers the components of chunks.

  • 3. Outside Parser

    State-of-the-art parsers that we want to accelerate.

    Parses the abstract representation of the input.

  • 4. Final Parse

    Need to collate the output from the two parsers.

    Results in a full dependency tree.

Chunker Implementation

We use the neural sequence toolkit NCRF++ developed by Yang and Zhang, 2018.

  • Features: POS tags

  • Character embedding dimensions: 30

  • Word embedding dimensions: 50

  • GloVe pretrained embeddings

  • Character layer: CNN (4 hidden layers, dim=50)

  • Word layer: BiLSTM (1 hidden layer, dim=200)

  • Inference later: CRF

Inside parser

  • Faster parsers

  • MaltParser with liblinear (Nivre et al., 2007)

  • Parsito (Straka et al., 2015)

  • Feedforward NN (Chen and Manning, 2014)

  • BIST parser (Kiperwasser and Goldberg, 2016)
